# 참고 구현 안내

이 starter 브랜치에는 참고 구현 코드나 비교 브랜치 안내를 포함하지 않습니다.

실습 중에는 아래 문서를 기준으로 직접 구현 흐름을 확인합니다.

- [이론 정리](./theory.md)
- [구현 가이드](./implementation.md)
- [체크리스트](./checklist.md)

## 구현 전에 확인할 질문

- Entity와 Response DTO는 왜 분리해야 하나요?
- Repository가 생기면 Service의 책임은 어떻게 달라지나요?
- 수정과 삭제도 같은 계층 흐름으로 설명할 수 있나요?
- 이번 단계에서 Validation과 예외 처리를 깊게 다루지 않는 이유는 무엇인가요?

## 멘토 안내

참고 구현 비교는 수업 운영 단계에서만 별도로 다룹니다. starter 브랜치에서는 비교용 코드, 비교 브랜치명, 완성 흐름 해설을 노출하지 않습니다.
