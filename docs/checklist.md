# 안전한 요청 처리 체크리스트

## 학생 체크리스트

- [ ] 요청 DTO와 응답 DTO를 왜 분리하는지 설명할 수 있습니다.
- [ ] `@Valid`와 `@NotBlank`가 어디서 동작하는지 말할 수 있습니다.
- [ ] 빈 제목 요청을 보내서 `VALIDATION_ERROR` 응답을 직접 확인했습니다.
- [ ] 없는 게시글 조회 요청을 보내서 `POST_NOT_FOUND` 응답을 직접 확인했습니다.
- [ ] 실패 응답을 같은 구조로 돌려주는 이유를 설명할 수 있습니다.

## 강사 / PPT 체크리스트

- [ ] 요청 DTO -> Validation -> Service -> ExceptionHandler 흐름 그림이 있는가
- [ ] 정상 요청과 실패 요청을 같은 화면에서 비교 시연할 수 있는가
- [ ] 검증 실패와 비즈니스 예외 차이를 학생이 말로 설명하게 할 수 있는가
- [ ] `ErrorResponse`가 왜 필요한지 코드와 응답 예시를 함께 보여줄 수 있는가
- [ ] 다음 시퀀스의 회원가입/로그인 입력 검증으로 자연스럽게 연결할 수 있는가
