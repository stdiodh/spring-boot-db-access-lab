# Google OAuth2 로그인 체크리스트

## 학생 체크리스트

- [ ] 자체 로그인과 Google 로그인의 차이를 설명할 수 있습니다.
- [ ] Google 응답에서 `email`, `sub`를 왜 읽는지 설명할 수 있습니다.
- [ ] 기존 OAuth 사용자 / 기존 로컬 사용자 / 신규 사용자를 어떻게 나누는지 설명할 수 있습니다.
- [ ] OAuth 로그인 성공 후 `auth-demo.html` redirect 결과를 직접 확인했습니다.
- [ ] 이번 시퀀스가 SMTP 기반 계정 복구와는 다른 흐름이라는 점을 설명할 수 있습니다.

## 강사 / PPT 체크리스트

- [ ] Google 로그인 -> 사용자 정보 읽기 -> 사용자 연결 -> JWT 발급 흐름 그림이 있는가
- [ ] `providerId`와 `email` 기준 분기를 코드로 보여줄 수 있는가
- [ ] 신규 사용자와 기존 사용자 시나리오를 각각 시연할 수 있는가
- [ ] `auth-demo.html`에서 OAuth 성공 redirect 결과를 보여줄 수 있는가
- [ ] 이후 SMTP 기반 계정 복구는 별도 레포로 분리할 계획임을 명확히 설명할 수 있는가
