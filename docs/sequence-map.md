# 시퀀스 맵

## 이 레포 안의 성장 흐름

### 02. 영속성 저장과 계층 분리

- 메모리 CRUD를 MySQL 기반 CRUD로 바꿉니다.
- `Controller -> Service -> Repository -> DB` 흐름을 익힙니다.

### 03. 안전한 요청 처리

- 요청 DTO와 응답 DTO를 나눕니다.
- Validation과 전역 예외 처리로 실패 응답을 통일합니다.

### 04. 인증과 JWT

- 회원가입과 로그인 흐름을 만듭니다.
- 로그인 이후 토큰으로 현재 요청 사용자를 구분합니다.

### 05. Google OAuth2 로그인

- Google 로그인 성공 후 사용자 정보를 읽습니다.
- 기존 사용자와 신규 사용자를 우리 서비스 기준으로 다시 연결합니다.
- 성공 결과를 JWT와 redirect 응답으로 정리합니다.

## 왜 이 순서인가

인증은 단독으로 뜨는 기능이 아니라,
이미 만들어 둔 저장 구조와 요청 검증 위에 얹혀야 자연스럽습니다.

그래서 이 레포는 아래 순서를 고정합니다.

1. 먼저 저장 구조를 만든다.
2. 그 다음 요청을 안전하게 만든다.
3. 그 위에 로그인과 JWT를 붙인다.
4. 그 다음 외부 로그인 확장 흐름을 붙인다.

이 순서를 지켜야 학생이 "왜 이 기능이 지금 추가되는지"를 단계별로 이해할 수 있습니다.
