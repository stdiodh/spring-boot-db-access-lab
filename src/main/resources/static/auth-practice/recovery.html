Receiver-side evidence

200 다음은 수신자 쪽에서 확인합니다.

SMTP accepted ≠ inbox

서버는 SMTP 요청 수락까지만 알 수 있습니다. 받은편지함 배치와 스팸 판정은 아래 증거를 직접 확인하세요.

  1. 받은편지함·프로모션·스팸함에서 제목과 발신자를 검색합니다.
  2. Gmail의 원본 보기에서 SPF·DKIM·DMARC 결과를 확인합니다.
  3. From·Return-Path·mailed-by·signed-by가 인증한 발신자와 정렬되는지 비교합니다.
  4. 로컬 링크 검증은 Mailpit으로, 실제 전달성 검증은 공개 HTTPS reset URL로 분리합니다.

Public response contract

200은 SMTP 수락, 4xx는 처음 멈춘 경계입니다.

  • SMTP 요청 수락200 RECOVERY_MAIL_SENT
  • 계정 없음·LOCAL 비밀번호 없음422 RECOVERY_MAIL_NOT_SENT
  • 재요청 cooldown429 RECOVERY_MAIL_COOLDOWN
  • 앱 비밀번호 인증 실패424 RECOVERY_MAIL_AUTHENTICATION_FAILED
  • SMTP 연결·전송 실패424 RECOVERY_MAIL_DELIVERY_FAILED

다른 실습 페이지로 이동하면 메모리의 reset token이 사라집니다. 필요하면 재설정 메일 링크를 다시 여세요.
