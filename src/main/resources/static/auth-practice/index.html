직접 확인할 흐름 · signup → login → /auth/me → POST /posts

내 계정이 요청의 신원이 되고,
게시글의 작성자가 됩니다.

이 실습의 아이디는 email입니다. 계정을 만든 뒤 같은 정보로 로그인하면, 서버가 보호 API에서 확인한 email을 게시글 작성자로 기록하는 데까지 한 흐름으로 보여줍니다.

Same-origin API 현재 서버

Authentication receipt

Credential이 Principal이 되는 세 지점

  1. 1 POST /auth/signup 계정 생성 입력 대기
  2. 2 POST /auth/login Token 발급 아직 token 없음
  3. 3 GET /auth/me 신원 확인 아직 Authentication 없음

입력

로그인 ID와 비밀번호

password는 trim하지 않습니다

회원가입과 로그인에 같은 email을 사용합니다.

계정 생성 시 8~64자, 로그인 시 최대 64자입니다.

email과 password를 직접 입력한 뒤 계정을 만들거나 로그인하세요.
